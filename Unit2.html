What is DOM

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page. A web page is a document that can be either displayed in the browser window or as the HTML source. In both cases, it is the same document but the Document Object Model (DOM) representation allows it to be manipulated. As an object-oriented representation of the web page, it can be modified with a scripting language such as JavaScript.

Doctype in HTML

In HTML, `<!DOCTYPE html>` is a declaration that specifies the version of HTML being used in the document. It's not an HTML tag; instead, it's an instruction to the web browser about how to interpret the document. The `<!DOCTYPE html>` declaration is used for HTML5 documents and is the simplest and most commonly used doctype. The purpose of the `<!DOCTYPE html>` declaration is to ensure that the web browser renders the document in standards mode, which helps ensure consistent rendering across different browsers. Before HTML5, there were various doctype declarations for different versions of HTML.

Charector Encoding

Character encoding is a system that associates each character in a character set with a unique numerical code. It allows computers to represent and manipulate text in various languages and scripts. There are several character encoding standards, each defining a mapping between characters and their corresponding numerical codes. Here are some common character encoding standards:

  1. ASCII (American Standard Code for Information Interchange): ASCII is one of the oldest and simplest character encoding standards. It uses 7 bits to represent 128 characters, including English letters, digits, punctuation marks, and control characters.
  2. ISO-8859 (Latin): The ISO-8859 series includes several character encoding standards, each designed for specific languages or regions. For example, ISO-8859-1 covers Western European languages.
  3. UTF-8 (Unicode Transformation Format 8-bit): UTF-8 is a variable-width character encoding standard that is capable of representing all Unicode characters. It uses 8-bit code units and can encode characters from various languages and scripts, making it the most commonly used character encoding on the web.
  4. UTF-16 (Unicode Transformation Format 16-bit): UTF-16 is another variable-width character encoding standard for Unicode. It uses 16-bit code units and can represent the entire Unicode character set.
  5. UTF-32 (Unicode Transformation Format 32-bit): UTF-32 is a fixed-width character encoding standard for Unicode. It uses 32-bit code units for each character and can represent the entire Unicode character set.
